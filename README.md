## What it is

Transponder is a self-hosted RSS reader built as a full-stack portfolio project. It aggregates feeds from 22 categories, persists bookmarks per authenticated user, and generates AI summaries on demand — all without sending your reading habits to a third party.

The AI layer is provider-agnostic: run [Ollama](https://ollama.com) locally for zero cost, or swap in [Groq](https://console.groq.com) (free tier) for a cloud deployment.

## Features

- **Authentication** — magic link email and GitHub OAuth via Supabase Auth
- **3-column reader** — categories / sources / articles, fully responsive (mobile Sheet drawer)
- **Per-user bookmarks** — stored in Postgres with Row Level Security, synced across devices
- **AI summaries** — 3-bullet-point digest generated by any OpenAI-compatible LLM, cached in DB
- **87 curated feeds** — 22 categories (EN + FR), recoverable from `supabase/seed_feeds_recovered.sql`
- **Docker-ready** — multi-stage image + Compose stack with Ollama as a sidecar service
- **Dual deployment** — Vercel (serverless) or Docker (node-server), same codebase

## Getting started

### Prerequisites

- A [Supabase](https://supabase.com) project (free tier is enough)
- One of:
  - **Docker** (recommended for local dev)
  - **Node.js 22+** (manual setup)

---

### Path A — Docker Compose (includes Ollama)

```bash
# 1. Clone
git clone https://github.com/your-username/transponder.git
cd transponder

# 2. Copy and fill environment variables
cp .env.example .env
# Edit .env — see Environment variables section below

# 3. Run the DB migrations in Supabase SQL Editor
#    supabase/migrations/001_initial.sql  (schema + RLS)
#    supabase/seed_feeds_recovered.sql   (87 feeds)

# 4. Start the stack
docker compose up --build

# 5. Pull the AI model (first run only)
docker compose exec ollama ollama pull llama3.2:3b
```

App available at `http://localhost:3000`.

> **Note:** with Docker Compose, `NUXT_OPENAI_BASE_URL` is automatically overridden to `http://ollama:11434/v1` (internal Docker network). You do not need to change it in `.env`.

---

### Path B — Manual setup

```bash
# 1. Clone and install
git clone https://github.com/your-username/transponder.git
cd transponder
npm install   # requires Node.js 22+

# 2. Environment variables
cp .env.example .env
# Edit .env with your values

# 3. Supabase — run in SQL Editor
#    supabase/migrations/001_initial.sql
#    supabase/seed_feeds_recovered.sql

# 4. Start Ollama (separate terminal)
ollama serve
ollama pull llama3.2:3b

# 5. Start the dev server
npm run dev
```

App available at `http://localhost:3000`.

## Environment variables

Create a `.env` file from `.env.example`:

| Variable | Description | Example |
|----------|-------------|---------|
| `NUXT_PUBLIC_SUPABASE_URL` | Supabase project URL | `https://xxx.supabase.co` |
| `NUXT_PUBLIC_SUPABASE_KEY` | Supabase anon/public key | `eyJhbGci...` |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key (server only) | `eyJhbGci...` |
| `NUXT_OPENAI_BASE_URL` | OpenAI-compatible API base URL | `http://localhost:11434/v1` |
| `NUXT_OPENAI_API_KEY` | API key (`ollama` for Ollama, real key for Groq) | `ollama` |
| `NUXT_AI_MODEL` | Model name | `llama3.2:3b` |

### Switching to Groq (free, cloud)

```env
NUXT_OPENAI_BASE_URL=https://api.groq.com/openai/v1
NUXT_OPENAI_API_KEY=gsk_your_key_here
NUXT_AI_MODEL=llama-3.1-8b-instant
```

Get a free key at [console.groq.com](https://console.groq.com) — no credit card required.

## Database setup

Run these two SQL files in order in your Supabase project's SQL Editor:

```
supabase/migrations/001_initial.sql   — tables, indexes, RLS policies
supabase/seed_feeds_recovered.sql     — 22 categories + 87 RSS feeds
```

The schema creates:

| Table | Description |
|-------|-------------|
| `categories` | Feed categories (public read) |
| `feeds` | RSS feed URLs per category (public read) |
| `user_bookmarks` | Per-user bookmarked categories (RLS) |
| `article_summaries` | Cached AI summaries per user (RLS) |

## Deployment

### Vercel (recommended)

```bash
# 1. Push to GitHub and connect the repo in vercel.com
# 2. Add all 6 environment variables in Vercel dashboard (Settings > Environment Variables)
#    Use Groq values for NUXT_OPENAI_BASE_URL / KEY / MODEL
# 3. Add your Vercel domain to Supabase Auth redirect URLs:
#    Authentication > URL Configuration > Redirect URLs
#    https://your-app.vercel.app/auth/confirm
```

Vercel auto-detects Nuxt and sets `NITRO_PRESET=vercel` — no additional config needed.

### Docker (self-hosted)

```bash
docker build -t transponder .
docker run -p 3000:3000 --env-file .env transponder
```

Or with Compose (includes Ollama):

```bash
docker compose up -d
```
